# Domain-Specific Reasoning Mappings

This reference provides quick access to the most relevant reasoning types for specific domains and contexts.

## Software Engineering

### Primary Reasoning Types
1. **Algorithmic Reasoning** - Step-by-step procedures, complexity analysis
2. **Systematic Reasoning** - Structured approaches, methodical testing
3. **Decompositional Reasoning** - Breaking down complex systems
4. **Diagnostic Reasoning** - Debugging, troubleshooting
5. **Constraint-Based Reasoning** - Working within technical limitations

### Secondary Reasoning Types
- **Comparative** - Technology selection, algorithm choice
- **Optimization** - Performance tuning, resource efficiency
- **Formal** - Type systems, verification
- **Systems** - Architecture design, distributed systems

### Common Tasks → Reasoning Types

| Task | Primary Reasoning | Supporting Reasoning |
|------|------------------|---------------------|
| Code Review | Critical, Analytical | Evidential, Evaluative |
| Debugging | Diagnostic, Abductive | Causal, Deductive |
| Architecture Design | Systems, Strategic | Constraint-based, Comparative |
| Algorithm Selection | Comparative, Quantitative | Optimization, Mathematical |
| Testing | Systematic, Evidential | Deductive, Statistical |
| Refactoring | Analytical, Decompositional | Systematic, Evaluative |
| Performance Tuning | Optimization, Quantitative | Evidential, Comparative |
| API Design | Systematic, Pragmatic | Conceptual, Game-theoretic |

## Cross-Domain Patterns

### Problem Types → Reasoning Types

**Well-Defined Problems** (clear goals, known solutions)
→ Algorithmic, Systematic, Deductive

**Ill-Defined Problems** (unclear goals, unknown solutions)
→ Creative, Divergent, Abductive, Exploratory

**Diagnostic Problems** (something wrong, find cause)
→ Diagnostic, Abductive, Causal, Evidential

**Optimization Problems** (improve performance)
→ Quantitative, Optimization, Comparative, Mathematical

**Strategic Problems** (competition, future planning)
→ Strategic, Game-theoretic, Scenario, Adaptive

**Learning Problems** (generalize from examples)
→ Inductive, Statistical, Pattern-recognition

**Explanation Problems** (understand why)
→ Abductive, Causal, Mechanistic, Theoretical

## Context-Specific Adjustments

### Time Pressure
- Prioritize: Heuristic, Intuitive, Pragmatic
- Avoid: Exhaustive analysis, optimization

### High Stakes
- Prioritize: Systematic, Evidential, Critical
- Avoid: Hasty decisions, single perspective

### Novel Situations
- Prioritize: Abductive, Creative, Analogical
- Avoid: Pure reliance on past patterns

### Uncertain Information
- Prioritize: Probabilistic, Bayesian, Provisional
- Avoid: Overconfident conclusions

### Team Decision
- Prioritize: Dialectical, Integrative, Communicative
- Avoid: Individual optimization ignoring others

### Continuous Improvement
- Prioritize: Adaptive, Evidential, Metacognitive
- Avoid: Fixed approaches, ignoring feedback

## Skill Development Pathways

### Beginner Level
Focus on:
- Deductive (clear logic)
- Inductive (pattern recognition)
- Comparative (evaluating options)
- Systematic (structured approaches)

### Intermediate Level
Add:
- Abductive (generating explanations)
- Strategic (planning)
- Diagnostic (troubleshooting)
- Evidential (weighing evidence)

### Advanced Level
Master:
- Game-theoretic (multi-agent reasoning)
- Bayesian (belief updating)
- Systems (holistic understanding)
- Meta-reasoning (reasoning about reasoning)

### Expert Level
Integrate:
- Adaptive (flexible approach selection)
- Integrative (synthesis across modes)
- Domain-specific (specialized applications)
- Reflexive (real-time adjustment)

## Conclusion

Different domains emphasize different reasoning types, but effectiveness comes from:

1. **Knowing your domain's primary reasoning types**
2. **Recognizing when to use secondary types**
3. **Combining types appropriately for complex problems**
4. **Adapting based on context (time, stakes, uncertainty)**
5. **Continuously improving through reflection**

Use this reference to quickly identify the most relevant reasoning approaches for your specific domain and context.
