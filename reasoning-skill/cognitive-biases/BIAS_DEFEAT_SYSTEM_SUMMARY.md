### 3. Practical Detection Protocols

**10 Battle-Tested Techniques:**

1. **Pre-Mortem Analysis**
   - Assume failure, work backward
   - Identify preventable failure modes
   - Build mitigation strategies

2. **Red Teaming**
   - Institutionalized adversarial review
   - Structured criticism
   - Devil's advocate roles

3. **Decision Journals**
   - Document reasoning before outcomes
   - Defeats hindsight bias
   - Enables calibration

4. **Consider the Opposite**
   - Force consideration of opposite conclusion
   - Seek disconfirming evidence
   - Bayesian updating

5. **Reference Class Forecasting**
   - Use similar past cases
   - Base rates over inside view
   - Statistical adjustment

6. **Think-Aloud Protocols**
   - Externalize reasoning
   - Catch logical jumps
   - Identify assumptions

7. **Anonymous Input**
   - Reduce social pressure
   - Defeat groupthink
   - Enable honest feedback

8. **Structured Processes**
   - ADRs, post-mortems, checklists
   - Systematize debiasing
   - Make it default

9. **Outside View**
   - Step back from specifics
   - Industry benchmarks
   - Expert consensus

10. **Prospective Hindsight**
    - "Imagine this failed, why?"
    - Anticipate problems
    - Build safeguards

### 5. Bias-Specific Detection Signals

**Each bias has unique "red flags":**

**Confirmation Bias Red Flags:**
- ‚ö†Ô∏è Only finding supporting evidence
- ‚ö†Ô∏è Quick dismissal of contrary evidence
- ‚ö†Ô∏è Interpreting ambiguous data as supportive
- ‚ö†Ô∏è Feeling defensive when challenged

**Availability Heuristic Red Flags:**
- ‚ö†Ô∏è Citing recent or dramatic examples
- ‚ö†Ô∏è "I know someone who..." as evidence
- ‚ö†Ô∏è Influenced by news coverage
- ‚ö†Ô∏è Ignoring statistical base rates

**Sunk Cost Fallacy Red Flags:**
- ‚ö†Ô∏è Justifying by past investment
- ‚ö†Ô∏è "We've come too far to quit"
- ‚ö†Ô∏è Emotional attachment to failing project
- ‚ö†Ô∏è Ignoring opportunity costs

**Overconfidence Red Flags:**
- ‚ö†Ô∏è Narrow confidence intervals
- ‚ö†Ô∏è Frequently surprised by outcomes
- ‚ö†Ô∏è "I'm sure" without uncertainty
- ‚ö†Ô∏è Underestimating time/difficulty
